# markbook DSL Syntax Reference

This document describes every construct the markbook Markdown DSL supports. A `.md` file using this syntax is compiled into a styled Jupyter Notebook via `markbook build`.

## Frontmatter

A YAML block delimited by `---` at the very top of the file. It configures notebook metadata and renders a title cell.

```yaml
---
title: "My Notebook"
kernel: python3
author: "Your Name"
---
```

| Field    | Required | Default   | Description                                      |
| -------- | -------- | --------- | ------------------------------------------------ |
| `title`  | no       | `""`      | Rendered as `# Title` in the first markdown cell |
| `kernel` | no       | `python3` | Jupyter kernel name (`python3`, `r`, `julia`)    |
| `author` | no       | `""`      | Rendered below the title as `Name: Author`       |

**Rules:**

- Must be the very first thing in the file (line 0)
- Opening and closing `---` must be on their own line
- Parsed with PyYAML (`yaml.safe_load`)

## Headings (Chapters)

Standard Markdown headings at levels 2–4 with optional anchor IDs:

```markdown
## 1. Chapter Title {#chapter1}

### 1.1 Sub-chapter {#chap1_1}

#### 1.1.1 Section {#chap1_1_1}
```

| Level  | Styling                                            |
| ------ | -------------------------------------------------- |
| `##`   | Large (1.6em), colored `#2E86AB`, 4px left border  |
| `###`  | Medium (1.3em), colored `#2E86AB`, 3px left border |
| `####` | Small (1.1em), colored `#555`, no border           |

**Anchor IDs:**

- Optional, specified as `{#anchor_id}` at the end of the heading text
- If omitted, an anchor is auto-generated by slugifying the heading text
- Used by `[TOC]` to create clickable links

**Rules:**

- Only levels 2, 3, and 4 are recognized (`##`, `###`, `####`)
- `#` (h1) is NOT treated as a chapter heading — it passes through as regular markdown
- Each heading becomes its own markdown cell with inline HTML styling

## Table of Contents

Place `[TOC]` on its own line to auto-generate a linked table of contents:

```markdown
[TOC]
```

**Rendering:**

- Generates a markdown cell titled "Gliederung"
- `##` headings → bold top-level links: `* **[Title](#anchor)**`
- `###` headings → indented links: `    * [Title](#anchor)`
- `####` headings → double-indented links: `        * [Title](#anchor)`

**Rules:**

- Must be on its own line, exactly `[TOC]` (no extra whitespace around it after strip)
- References ALL headings in the document, including those that appear before it
- Headings without explicit anchors get auto-generated slugified anchors

## Code Cells

Fenced code blocks with a recognized language tag become Jupyter code cells:

````markdown
```python
import pandas as pd
df = pd.read_csv("data.csv")
df.head()
```
````

**Supported languages:**

`python`, `bash`, `sql`, `r`, `julia`, `sh`, `javascript`, `typescript`, `java`, `c`, `cpp`, `go`, `rust`

**Rules:**

- The language tag must be a single word directly after the opening ` ``` ` (no space)
- Language matching is case-insensitive
- Fenced blocks with an **unrecognized** or **missing** language tag become **markdown** cells (not code cells)
- The closing ` ``` ` must be on its own line
- Unclosed fenced code blocks raise a `MarkbookSyntaxError`

## Section Dividers

A horizontal rule `---` on its own line (not at the top of the file) becomes a styled `<hr>`:

```markdown
---
```

**Renders as:**

```html
<hr style="border: none; border-top: 2px solid #dee2e6; margin: 20px 0;" />
```

**Rules:**

- Only recognized when NOT on line 0 (line 0 `---` starts frontmatter)
- The line must contain exactly `---` (after stripping whitespace)

## Regular Markdown

Everything else — paragraphs, bold, italic, lists, tables, blockquotes, inline code, images, links — passes through as-is into markdown cells.

**Rules:**

- Consecutive non-special lines are grouped into a single markdown cell
- Empty lines within regular text are preserved
- A special construct (heading, code fence, `[TOC]`, `---`) flushes the current markdown buffer into its own cell

## Complete Example

````markdown
---
title: "Analysis Report"
kernel: python3
author: "Data Team"
---

[TOC]

---

## 1. Introduction {#intro}

This report analyzes the dataset.

### 1.1 Goals {#goals}

- Classify outcomes
- Visualize distributions

---

## 2. Analysis {#analysis}

```python
import pandas as pd
df = pd.read_csv("data.csv")
df.head()
```
````

Results are shown above.

---

```

This produces a notebook with:
1. Title cell ("# Analysis Report" + "Name: Data Team")
2. Table of contents cell with links to all headings
3. Styled `<hr>` divider
4. Styled `## 1. Introduction` heading cell
5. Markdown cell with the paragraph
6. Styled `### 1.1 Goals` heading cell
7. Markdown cell with the bullet list
8. Styled `<hr>` divider
9. Styled `## 2. Analysis` heading cell
10. Python code cell
11. Markdown cell with "Results are shown above."
12. Styled `<hr>` divider
```
